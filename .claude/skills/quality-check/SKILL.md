# Skill: quality-check — 班長エージェントによる品質チェック

## 概要
班長エージェントが各ステップで実施する3段階の品質チェック手順。
問題が見つかった場合は該当エージェントに差し戻し、最大2回まで修正を要求する。

---

## チェック① — リサーチ結果の確認

リサーチエージェントの出力JSONを受け取ったら以下を確認する。

| チェック項目 | 確認方法 |
|---|---|
| 既存記事との重複 | `en/` フォルダのファイル名一覧と照合 |
| slug の重複 | 同上 |
| キーワードの適切さ | ターゲット読者（外国人工場労働者）に合っているか |
| 3言語タイトルの存在 | JSON に title_en / title_tl / title_vi が含まれているか |

**NG時:** リサーチエージェントに差し戻し（理由を明示）
**OK時:** 3言語ライターに同時指示

---

## チェック② — 記事完成後の確認

3言語ライターが完了を報告したら、以下をコマンドで機械的に確認する。

```bash
# H2セクション数（5〜8個が合格）
grep -c "<h2" {lang}/{slug}.html

# 必須コンポーネントの存在（各1以上が合格）
grep -c "tip-box"     {lang}/{slug}.html
grep -c "warning-box" {lang}/{slug}.html
grep -c "key-table"   {lang}/{slug}.html

# style.cssパスの正しさ（1が合格）
grep -c "\.\./style\.css" {lang}/{slug}.html
```

追加確認（目視）:
- 日本語用語の初出にローマ字が付いているか
- 各 `{lang}/index.html` に新記事カードが追加されているか
- article-header / article-body / footer の構造が正しいか

**NG時:** 該当言語のライターのみ差し戻し（修正箇所を番号付きで指示）
**全言語OK時:** 投稿エージェントに指示

---

## チェック③ — push結果の確認

投稿エージェントの完了報告を受けたら以下を確認する。

| チェック項目 | 確認方法 |
|---|---|
| pushの成功 | 出力に `main -> main` が含まれているか |
| 3言語ファイルがコミットに含まれているか | `git log --stat -1` で確認 |

**NG時:** 投稿エージェントにエラー内容を渡して差し戻し
**OK時:** 作業レポートを出力

---

## 差し戻しルール

- 差し戻しは**各エージェントにつき最大2回**
- 2回修正後もNGの場合は**作業を停止してユーザーに報告**する
- 差し戻し時は修正箇所を具体的に指示する（「〇〇が足りない」「〇〇を直せ」）

---

## 作業レポートテンプレート

全チェック通過後、以下の形式で日本語レポートを出力する。

```
## 班長レポート

### 作成記事
- タイトル（EN）：{title_en}
- タイトル（TL）：{title_tl}
- タイトル（VI）：{title_vi}
- ファイル名：{slug}.html（en/ tl/ vi/ 各フォルダ）
- カテゴリ：{badge_category}

### 各エージェントの作業結果
- リサーチ：完了（差し戻し回数：{n}回）
- 英語ライター：完了（差し戻し回数：{n}回）
- タガログ語ライター：完了（差し戻し回数：{n}回）
- ベトナム語ライター：完了（差し戻し回数：{n}回）
- 投稿：完了（コミットハッシュ：{hash}）

### 品質チェック結果
- テーマ重複：なし
- H2セクション（EN / TL / VI）：{n} / {n} / {n}
- tip-box / warning-box / key-table：全言語適合
- style.cssパス：全言語確認済み
- GitHub push：成功

### 備考
{気になった点・次回への改善提案があれば記載、なければ「なし」}
```
